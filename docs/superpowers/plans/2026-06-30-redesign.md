# Redesign tomasbruckner.dev Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the legacy Next.js 10 / MUI personal site with a fast, bilingual (cs/en), minimal, SEO + AI-search-optimized static site built on Astro 5 + Tailwind 4, deployed to Cloudflare Pages.

**Architecture:** Single-page static site with sections (Hero/About, Lectures, FAQ, Contact). Zero JS by default; only four tiny vanilla-JS islands (language switch, theme toggle, mobile menu, copy-email). Content lives in typed i18n dictionaries; `cs` is the default locale at `/`, `en` at `/en/`. SEO via per-locale metadata, hreflang, JSON-LD (Person, ProfessionalService, VideoObject, FAQPage), sitemap, robots, and `llms.txt`.

**Tech Stack:** Astro 5.x, Tailwind CSS 4 (`@tailwindcss/vite`), TypeScript, `@fontsource-variable/inter`, `@astrojs/sitemap`, Vitest (pure-logic unit tests), Playwright (e2e smoke), GitHub Actions CI, Cloudflare Pages + Cloudflare Web Analytics.

## Global Constraints

- Astro `^5` (>=5.2 required for `astro add tailwind`). Tailwind CSS `^4` via `@tailwindcss/vite` — do NOT use the deprecated `@astrojs/tailwind` integration.
- Static output only (no SSR adapter). Build output directory: `dist/`.
- Site URL: `https://tomasbruckner.dev` (set as `site` in `astro.config.mjs`).
- Locales: `cs` (default, served at `/`) and `en` (served at `/en/`). Config: `routing: { prefixDefaultLocale: false }`.
- Zero-JS default. Interactivity ONLY in four islands: LangSwitch, ThemeToggle, mobile menu, CopyEmail. No React/Vue/Svelte runtime.
- Light theme default, but honor `prefers-color-scheme` on first visit; persisted user choice in `localStorage` key `theme` wins. Dark mode is class-based (`.dark` on `<html>`).
- Email (verbatim): `tomasbrucknermail@gmail.com`.
- Social links (verbatim): GitHub `https://github.com/tomasbruckner`, StackOverflow `https://stackoverflow.com/users/3323141/tomas-bruckner`, Medium `https://medium.com/@tomasbruckner`, LinkedIn `https://www.linkedin.com/in/tomasbruckner`.
- AI crawlers MUST be allowed in `robots.txt`: `GPTBot`, `ClaudeBot`, `PerplexityBot`, `Google-Extended`.
- Reuse existing assets from `public/static/` (favicons, manifest icons, `images/profile.jpg`).
- Each task ends with a passing verification (build / `astro check` / vitest / playwright) and a commit.

- [ ] **Step 3: Write the helper `src/i18n/util.ts`**

```ts
import { cs } from './cs';
import { en } from './en';

export type Lang = 'cs' | 'en';

const dictionaries = { cs, en } as const;

export function useTranslations(lang: Lang): typeof cs {
  return dictionaries[lang];
}

export function getAltLangUrl(lang: Lang): string {
  return lang === 'cs' ? '/en/' : '/';
}

export function altLang(lang: Lang): Lang {
  return lang === 'cs' ? 'en' : 'cs';
}
```

---

    data-id={youtubeId}

  >

    <span class="absolute inset-0 grid place-items-center bg-black/20 transition group-hover:bg-black/30">

      'click',

        const title = btn.getAttribute('data-title') ?? '';
        const iframe = document.createElement('iframe');
        iframe.width = '100%';
        iframe.height = '100%';
        iframe.title = title;
        iframe.allow =
          'accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture';

        iframe.style.border = '0';

        btn.replaceWith(iframe);
      },
      { once: true },
    );
  });
</script>

```markdown
# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Personal site for Tomáš Bruckner. Astro 5 + Tailwind 4, TypeScript, bilingual (cs default at `/`, en at `/en/`), static output, deployed to Cloudflare Pages. Single-page layout: Hero/About → Lectures → FAQ → Contact.

## Commands

    npm run dev         # dev server (http://localhost:4321)
    npm run build       # static build -> dist/
    npm run preview     # serve built output
    npm run check       # astro type check
    npm run test:unit   # vitest (pure logic: i18n, data integrity)
    npm run test:e2e    # playwright smoke (auto build+preview)

Run a single unit test: `npx vitest run tests/unit/i18n.test.ts`
Run a single e2e test: `npx playwright test -g "theme toggle"`

## Architecture

- **Zero-JS default.** Only four interactive islands, each a small `<script>` in its `.astro` component: `ThemeToggle`, `LangSwitch` (just a link), mobile menu (in `Nav`), `CopyEmail`. No UI-framework runtime.
- **i18n is type-driven.** `src/i18n/cs.ts` is the source of truth; `src/i18n/en.ts` is typed `typeof cs`, so a missing/renamed key fails `astro check`. `useTranslations(lang)` returns the dictionary; access keys as typed properties (`t.nav.about`), not string paths. Locale comes from the route, never a mutable global.
- **Content is data.** Videos in `src/data/lectures.ts` (id + i18n key), FAQ in `src/data/faq.ts`; titles/answers live in the dictionaries. Site constants (email, socials, URL, analytics token) in `src/consts.ts`.
- **SEO/AI-search.** `Base.astro` emits per-locale title/description, canonical, hreflang (cs/en/x-default), OpenGraph/Twitter, and JSON-LD (Person, ProfessionalService). `Lectures` emits one `VideoObject` per talk; `Faq` emits `FAQPage`. `public/robots.txt` allows AI crawlers (GPTBot, ClaudeBot, PerplexityBot, Google-Extended); `public/llms.txt` summarizes the site; sitemap via `@astrojs/sitemap`.
- **Theme.** Class-based dark mode (`.dark` on `<html>`). Inline `<head>` bootstrap in `Base.astro` applies the stored/`prefers-color-scheme` choice before paint; `ThemeToggle` persists to `localStorage.theme`.
- **Video performance.** `VideoCard` is a lite-embed: thumbnail + play button; the real `youtube-nocookie` iframe is injected only on click.

## Conventions

- Add/edit copy only in `src/i18n/{cs,en}.ts` (keep shapes identical).
- New section = new `.astro` in `src/components/`, take a `lang: Lang` prop, wire into both `src/pages/index.astro` and `src/pages/en/index.astro`.
- Dark styles via Tailwind `dark:` variant; accent color is `--color-accent` (defined in `src/styles/global.css`).

## Deploy

Cloudflare Pages, Astro preset, build `npm run build`, output `dist`. Push to `master` = production; PRs get previews. CI (`.github/workflows/ci.yml`) runs check + unit + build + playwright.
```
